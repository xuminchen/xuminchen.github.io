# Introduction

# 前言

随着人工智能领域的火起来，越来越多科研人员、工程师投身到AI这个巨大无比的深坑之中，而AI又可以更加细分，当下最火热的，无非是CV，NLP，还有推荐系统。而我，则是在NLP的浪潮中摸索的一员。在摸索过程中，遇到一些坑，记录下来，以供后面复习。

# What is NLP?

nlp 全名natural language processing, 包含了两个模块（两个部分），其中一个是NLU，即natural language understanding，而另一个叫NLG，即natural language generation.

NLU难处：

  • 多样性

  • 歧义性

  • 鲁棒性

  • 知识依赖

  • 上下文

NLG的六个步骤：

  1. 内容确定 Content Determination
  2. 文本结构 Text Structuring
  3. 句子聚合 Sentence Aggregation
  4. 语法化 Lexicalisation
  5. 参考表达式生成 Referring Expression Generation (REG)
  6. 语言实现 Linguistic Realisation

# How to figure out an NLP problem?

# Pipeline

  1. 原始文本(raw data)
  2. 分词(segmentation)
    • 包括中英文分词
  3. 清洗(cleaning)
    • 无用的标签
    • 特殊符号
    • 停用词(根据场景过滤)
    • 大小写转换
  4. 标准化(normalization)
    • 一般用在英文分词上，把多个单词合并成一个单词的过程
    • Stemming
    • Lemazation
  5. 特征提取(feature extraction)
  6. 建模(modeling)

在文本预处理一节中，我会稍微总结一下目前的预处理工作。而特征提取和建模部分则是稍微靠后的环节，特征提取是属于特征工程中的一部分，而建模则包括了机器学习建模和深度学习建模，稍后会讲到的LR和SVM等，都属于机器学习的范畴，而RNN等则是属于深度学习的模型。

# What is Language Model?

Language Model是用来判断一句话在语法上是否通顺的。

**Goal:**Compute the probability of a sentence or sequence of words.

我们通过每个词在Corpus(文本库、语料库)中出现的次数来表示这个词的概率，记作P(w)P(w)P(w)

我们有一句话：w1+w2+w3+w4+w5w_1+w_2+w_3+w_4+w5w​1​​+w​2​​+w​3​​+w​4​​+w5由5个词语组成，这一句话的概率就是P(w1,w2,w3,w4,w5)P(w_1,w_2,w_3,w_4,w_5)P(w​1​​,w​2​​,w​3​​,w​4​​,w​5​​)，我们可以用Chain Rule去计算：

P(w1,w2,w3,w4,w5)=P(w1)∗P(w2∣w1)∗P(w3∣w1,w2)∗P(w4∣w1,w2,w3)∗P(w5∣w1,w2,w3,w4)P(w_1,w_2,w_3,w_4,w_5)=P(w_1)*P(w_2|w_1)*P(w_3|w_1,w_2)*P(w_4|w_1,w_2,w_3)*P(w_5|w_1,w_2,w_3,w_4)P(w​1​​,w​2​​,w​3​​,w​4​​,w​5​​)=P(w​1​​)∗P(w​2​​∣w​1​​)∗P(w​3​​∣w​1​​,w​2​​)∗P(w​4​​∣w​1​​,w​2​​,w​3​​)∗P(w​5​​∣w​1​​,w​2​​,w​3​​,w​4​​)

但是这时候就会有一个问题，那就是在文本库中，4个词、5个词甚至更多的词并不一定在一句话中同时出现，这时候就会导致概率为0，这个就是数据的稀疏性问题。

因此为了解决数据稀疏性的问题，引入了Markov Assumption(马尔科夫假设)，即只考虑与当前单词最相近的前n个。比较常见的Markov Assumption有：

  • first order Markov Assumption - P(wi∣wi−1)P(w_i|w_{i-1})P(w​i​​∣w​i−1​​)，也就是n-gram模型中的bi-gram
  • second order Markov Assumption - P(wi∣wi−1,wi−2)P(w_i|w_{i-1},w_{i-2})P(w​i​​∣w​i−1​​,w​i−2​​)，也就是n-gram模型中的tri-gram
  • third order Markov Assumption - P(wi∣wi−1,wi−2,wi−3)P(w_i|w_{i-1},w_{i-2},w_{i-3})P(w​i​​∣w​i−1​​,w​i−2​​,w​i−3​​)，也就是n-gram模型中的fourth-gram

当然，这是在我们考虑到了单词顺序的情况下，当我们不考虑顺序的时候，则为uni-gram。

# How to evaluate a Language Model?

那么问题来了，我们该怎么评估语言模型的好坏呢？

在理想情况下，我们是这样评估的：

  1. 首先假设有两个语言模型A和B
  2. 选定一个特定的任务，比如Spell Correction(拼写纠错)
  3. 把两个语言模型都应用到此任务中
  4. 最后比较准确率，从而判断A和B的表现哪个更佳

但是，如果每次都要训练两个语言模型，或者说更多个的话，无疑会增加训练开销，因此我们提出了一个叫**Perplexity(困惑度)**的东西。

Perplexity is trouble or confusion resulting from complexity.

Perplexity的基本思想就是：给测试集的句子赋予较高概率值的语言模型较好，当训练完后，希望所得到的的句子可以得到更高的概率。

Perplexity=2−xPerplexity = 2^{-x}Perplexity=2​−x​​，其中xxx是average log likelihood。

举个例子，假如在bi-gram中,

LM=P(w1,w2,w3,w4,w5)=P(w1)∗P(w2∣w1)∗P(w3∣w2)∗P(w4∣w3)∗P(w5∣w4)LM = P(w_1,w_2,w_3,w_4,w_5)=P(w_1)*P(w_2|w_1)*P(w_3|w_2)*P(w_4|w_3)*P(w_5|w_4)LM=P(w​1​​,w​2​​,w​3​​,w​4​​,w​5​​)=P(w​1​​)∗P(w​2​​∣w​1​​)∗P(w​3​​∣w​2​​)∗P(w​4​​∣w​3​​)∗P(w​5​​∣w​4​​)

因为概率都是小于1的值，这样会导致语言模型的结果是一个非常非常小的数值，所以我们用log对数改变一下形式，另外，因为我们只是希望比较概率大小，并不在意这个概率数值是多少，加了log之后也不影响语言模型的单调性，因此

LM=logP(w1)+logP(w2∣w1)+logP(w3∣w2)+logP(w4∣w3)+logP(w5∣w4)LM = \log P(w_1)+\log P(w_2|w_1)+ \log P(w_3|w_2)+\log P(w_4|w_3)+ \log P(w_5|w_4)LM=logP(w​1​​)+logP(w​2​​∣w​1​​)+logP(w​3​​∣w​2​​)+logP(w​4​​∣w​3​​)+logP(w​5​​∣w​4​​)

一开始说过，语言模型的目的是计算句子的概率，概率越大，表示的是句子越通顺，我们令x=LM5x=\frac {LM}{5}x=​5​​LM​​，为了抵消log的影响，以2为底的指数形式是:2x2^x2​x​​，这也就是我们的语言模型的其他形式。不过我们这里求的是困惑度，希望困惑度越小越好，因此让x的值取负数，即2−x2^{-x}2​−x​​。

但是并不代表perplexity越小，语言模型越好。这个是非绝对的，要在实际的任务场景中验证才知道效果如何。

另外，在wiki和一些资料上看过，Perplexity中这个x的值是以Cross Entropy的形式出现的，但是在这里并不是，我个人理解就是，在无标签无监督的情况下，作为的一个评估标准，因此没有和其他label比较，所以不是以Cross entropy的形式出现。

# How to represent a word/sentence/character?

# One-Hot Representation

根据词典长度决定词向量的长度，词向量每一位都表示其中一个单词，若在文本中出现，则为1，否则为0。

# Boolean Representation

依然是根据词表长度决定，但是不同的是，One-Hot Representation中，每个词向量只表示一个单词，而在Boolean Representation中，每个向量表示的是一个文本，而向量中的每一个值则表示的是该词是否在文本中出现。同样，出现过一次或多次用1表示，没出现过用0表示。

# Count-Based Representation

依然采用向量的方式表达，向量的长度根据词表确定。每个向量表示的是一个文本，而向量中的每一个值则表示的是该词是否在文本中出现。同样，出现过一次或多次用表现的次数表示，没出现过用0表示。

要注意的是并不是出现的越多的就越重要，并不是出现的越少就越不重要。因此采用TF-IDF的方法去进行改善。

# TF-IDF Representation

tfidf(w)=tf(d,w)∗idf(w)tfidf(w)=tf(d,w)*idf(w)tfidf(w)=tf(d,w)∗idf(w)

tf(d,w)tf(d,w)tf(d,w)表示文档d中w的词频

idf(w)=logNN(w)idf(w)=\log \frac{N}{N(w)}idf(w)=log​N(w)​​N​​，其中N表示语料库中的文档总数，N(w)表示词语w出现在多少个文档中。

关于idf这一项的思想是考虑单词的重要性，一旦某个单词出现在很多句子中，则说明重要性较低；一旦出现的比较少，说明重要性比较高。log是防止这一项过大。

然而，以上的方法具有稀疏(sparsity)、文本相似度计算不佳等问题。因此，引入了分布式表达Distributed Representation这一概念。

# Distributed Representation

与传统的one-hot等表达不一样，那些文本表达方式是以向量形式存在，而向量的长度是根据文本库的词典总大小来确定的，如一个文本库的词典含有100个单词，那么这些文本表达向量的维度就为100；而分布式表达则不一样，分布式表达可以使向量中的每一个维度都有一个值，并且向量的维度并不等于词典的大小，而是可以人为设定。同样是100维的向量，分布式表达的表达空间远远比one-hot等表达的空间大小大得多。

什么是词向量呢？词向量从某种意义上表达的是单词的意思(meaning).并且在某种意义上，可以捕获单词的语义，使得在某个维度空间中，语义相近的单词会比较相近。
