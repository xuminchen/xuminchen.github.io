# 决策树系列

# Introduction

决策树是机器学习领域中比较基础也很重要的一个知识点，在最开始，人们通过if-else的判断语句，可以转换成一颗基于每一个特征或条件的树，但是这样生成的树很容易导致过拟合。因此人们在后续的漫长时间长河里，研究如何去构造一颗树，怎么去选择特征，及怎么防止过拟合做出了很大研究，其中Id3，C4.5，CART是非常重要且基础的研究成果。

除此之外，决策树还能被其他技术所采纳，生成更加广泛，更加高效，效果更好的技术，如Random Forest，GBDT等，以及它们的工程实现如XGBOOST，LightGBM等。

# Id3

基于奥卡姆剃刀原则的基础上，去决定一棵决策树，在同等效果下，选择更小的树。

奥卡姆剃刀原则就是，在具有同等效果的条件下，选择模型复杂度更小的作为我们的基础模型。

# 思想

信息熵越大，样本纯度越低。所以ID3以信息增益来度量特征选择带来的影响，并选择增益最大的特征进行分裂。

要注意的一点就是，在决策树的构造中，很多都是在信息论的基础上提出的，像信息熵这类的理论基础都是，因此这里采用的log计算都是以2为底的。

# 算法

自顶向下的贪婪搜索，遍历所有可能的决策树空间。

  1. 初始化所有可能的特征集合和数据集合
  2. 计算数据集合的信息熵和所有未选择特征的条件熵，来选择信息增益
  3. 更新数据集合和特征集合（也就是删除上一步决定使用的特征）
  4. 重复step2，3， 直到选择好后的数据属于同一类，这一类为分支的叶子结点

# 缺点

  1. id3算法并没有剪枝策略，容易过拟合
  2. 只能用于处理离散值
  3. 无法处理缺失值
  4. 更偏向于数量大的特征

# 剪枝策略

为了防止决策树过拟合，提出了一些剪枝方法，防止决策树过拟合。剪枝手段分两种，一种是预剪枝，另一种是后剪枝。预剪枝就是在决策树还没生成完全的时候就执行剪枝；而另一种是树已经生成后，再剪枝。

预剪枝常用策略：

  1. 当到达一定深度时
  2. 错误率到达可接受的范围
  3. 树的叶子节点数

后剪枝：

  1. 悲观剪枝

# C4.5

与id3算法相比，引入了悲观兼职策略作为一种兼职手段。而且采用了信息增益比作为新的划分标准，同时提供处理连续性特征的能力，同时也能够处理缺失值。

从这里我们可以看出，C4.5算法与id3算法相反，它对特征样本子集的数量越小，关注度越大。

另外，C4.5实际上采用的是一种启发式算法，先从候选集中找到信息增益率高于平均值的特征，再从中选择最大的

# 剪枝策略

C4.5用的是一种叫悲观剪枝的后剪枝策略，方法就是：用递归的方法，生成从满树到只剩下一个根节点的一个列表，从下往上针对每一个非叶子结点，评估一个最佳叶子结点去代替该子树是否有益，如果剪枝前后错误率保持变或下降，则可以被替换。

c4.5通过训练机上的错误分类来估计位未知的情况。

# 缺失值处理

C4.5算法提供了对缺失值处理的能力，但是有两个问题：

  1. 如何计算确实的特征增益率
  2. 如何划分信息缺失点

关于第一个问题，可以采用无缺失的样本自己占用总样本子集数的比重来折算

而第二个问题，就是样本以不同的概率划分到不同的结点中

# 缺点

  1. 生成的可能是一棵多叉树，会增加计算开销
  2. 只能用于分类任务
  3. 大量耗时的熵对数计算
  4. 连续型特征排序时耗时
  5. 不能超过内存

# CART

id3和c4.5算法为了从数据集中挖掘出更多的信息，导致构建的树的规模比较大，与它们相比，CART简化了树的规模，提高了生成决策树的效率。

CART全名是Classification And Logistic Tree，从名字可以看出，CART是提供处理回归任务的能力的。

# 思想

无限的二叉递归划分，无停止准则，输入和预测特征即可以是连续型，也可以是离散型的。

在树构建完后，用单独的测试集评估每棵剪枝树的性能。

# 缺失值处理

采用了替代划分（Surrogate Splits），先在非确实的数据上划分，然后根据确实的比例呈上相应技术降低存在缺失值的gini，然后与其他分裂怎一进行比较。

原则:使其分裂的效果与最佳分裂的特征效果相似，即分裂的误差最小。

  1. 在缺失的是最大增益的特征的情况下，遍历剩余的特征，若还有缺失，则无视，仅考虑完整的特征，选择其中能与最佳增益的缺失特征在分裂之后最相近的特征。
  2. 如果事先设定一定的标准，仅仅选择差异性在一定范围内的特征作为代理特征，从而导致没有特征与最佳分裂的特征效果相似，或所有都存在缺失，则默认进入个数最大的叶子结点。

# 集成学习

为什么要提出集成学习？

  1. 单一的模型可能无法找到最优解，而用多个模型可以去逼近
  2. 假设可以找到最优解，但是由于计算能力的限制，导致无法直接找到最优解，只能找到了次优解，而集成学习可以弥补这个不足
  3. 单个学习器可能无法选择出最好的，所以把多个学习器放到一起用。

# Bias and Variance

偏差和方差是两个衡量模型效果的重要角度。我们希望找到一个偏差和方差都小的模型。

Bias指的是预测值与真实值之差，如果想要提高，则需要更复杂的模型，但是这样就会导致overfitting。

Variance指的是预测值作为随机变量的离散程度，如果想要减小，则需要简化模型，而这样就会导致high bias，以及可能出现underfitting

集成学习可以分为以下三类

# Bagging

全称Bootstrap aggregating, 该方法采用了有放回的抽样方法(0.632自助法)，通过多个及学习器并行学习，最后通过投票法来综合所有学习器作为最终结果。

因为bagging是并行学习的，学习器之间具有弱相关性，为了让每个学习器的效果都好，所以我们采用了强学习器（低偏差，高方差）。如果采用了弱学习器（高偏差，低方差），会导致在投票的时候出现高Bias。

另外，每个强学习器的权重是1m\frac {1}{m}​m​​1​​，期望近似相等。但方差大于整体模型的方差，当且仅当学习器的相关性等于1时，两者方差才相等。随着基学习器数量上升，模型整体的方差减小，从而预防模型overfitting，提高了accuracy。但是并不是没有限制的，就算基学习器数量增多，但是预测的结果有限，最终会达到某一点处后便不再提升。

# Boosting

Boosting是通过多个基学习器串行学习训练的，每个学习器都是在前一个学习期的基础上学习，最后通过加权法将结果综合起来，作出prediction。

因为每一个学习器都是在前一个学习器的基础上训练的，所以模型之间具有强相关性，因此采用的是弱学习器，整体的方差等于基学习器的方差。如果用强学习器，则方差大，容易overfitting。

Boosting采用基于贪心策略的前向算法，整体期望由基学习器的期望累加，基学习器的数量越多越准确。

# Stacking

先训练好所有的基学习器，这些基学习器都是强学习器，然后用这些学习器都对样本进行预测，然后将预测的结果作为特征，得到新的样本，最后基于这些新的样本去训练模型。

# Random Forest

随机森林是在Bagging的思想上，以决策树作为基学习器，多棵树构成的模型。另外，在树的生成时，引入了随机特征选择。

# 算法流程

  1. 随机选择样本（有放回地随机抽样，与Bagging一样，采用Bootstrap自助采样）
  2. 随机选择特征，在每个结点的分裂过程中，都是随机选择特征的
  3. 构建决策树
  4. 随机森林投票产出结果。

# 优点

  1. 因为随机性导致Bias上升，但是Bagging的平均特性，使得Variance下降。因此就算不用剪枝策略也不会导致overfitting。
  2. 并行计算，多棵树同时计算，能够同时处理大数据集
  3. 能够处理高维特征，而且是自助特征选择

# AdaBoost

全称Adaptive Boosting

# 思想

基于前一个基学习器分类，错误的样本会被加强（加权操作，使得下一个学习器更关注这些样本）。加权后的全体样本再用来训练下一个基学习器。同时，每一轮训练加入一个新的基学习器，知道满足设置的错误率(阈值)，或达到了最大的训练轮数。

# 算法步骤

  1. 初始化所有样本的权值，此时所有样本的权值都一样
  2. 开始训练弱学习器，如果分类正确，该样本的权值会减小，否则的话会增大。并用更新过的样本去训练下一个基学习器。
  3. 将所有基学习器组合成一个强学习器，在各个弱学习器训练完成后，加大分类误差率小的弱学习器的权重，降低错误率大的学习器的权重。

# GBDT

全称Gradient Boosting Decision Tree，梯度提升决策树，从名字很容易可以看出，这里采用的是一种用梯度替代的方法。

GBDT采用的是CART 回归树，因为每次迭代的时候拟合的是梯度值，而且梯度值是连续的。另外值得一提的是，回归树的重点是用均方误差寻找最佳分裂点，包含了特征可选所有的值，分裂时会穷举每个特征的threshold，并找到最小均方误差的值。核心是累加所有树的结果为最终的预测结果。

GBDT可以分成三个部分：GB(Gradient Boosting)，DT(Decision Tree) 和 Shrinkage.

# Shrinkage

缩减

思想是：每次一小步渐渐地逼近比每次一大步快速逼近更容易避免overfitting，也就是说，它并不是完全信任每一棵残差树的。

Shrinkage不是直接修复残差，实际上为每一棵树设置了一个权重，作为最后加权平均的权重。当权重减小时，f(x)f(x)f(x)的值就会增大。

Ft(x)=Ft−1(x)+μft(x)F_t(x)=F_{t-1}(x)+\mu f_t(x) F​t​​(x)=F​t−1​​(x)+μf​t​​(x)

# 与AdaBoost的不同

  • 迭代方面：Adaboost是更关注错误的样本的，对错误的样本，它的权重是增大的，对正确的样本，它的权重就会下降；而GBDT的残差计算是变相地增大了被错误分类的样本的权重，而正确的样本则趋于0.
  • 损失函数方面：Adaboost采用的是指数损失函数，而GBDT采用的是Huber、绝对损失、MSE。

# 优缺点

Pros

  1. 自动特征组合，拟合非线性的数据
  2. 可以灵活处理各类型的数据

Cons

  1. 对异常值敏感

# XGBOOST

XGBoost是GBDT算法的工程实现。

XGBOOST有四个问题：

首先是如何构造目标函数

其次如何近似目标函数呢？因为XGBoost的优化问题不是一个连续型的优化问题，而是一个离散型的优化问题。因此它的优化问题要复杂许多，所以xgboost要对目标函数进行一个简化，这里采用的是泰勒展开式。

然后是如何把树的结构引入到目标模型， 因为目标函数中是要去表示一棵树的，所以我们需要通过将树的结构参数化，通过一些参数去描述一棵树。

因为这是一个NP-Hard problem，所以我们要使用Greedy Algorithm去优化。

# 寻找树的形状

首先，用最笨的方法——Brute-Force Search暴力法，遍历所有的树，然后找到最小的那棵树。但是，这是不可行的，因为它的复杂度会呈指数型增长。

所以我们用Greedy Search贪心算法去寻找出来，参考信息增益的做法，生成一棵新树的前后的obj的变化。那么怎么去找到这个最好的切分点（split）呢？最大化objOld−objNewobj_{Old}-obj_{New}obj​Old​​−obj​New​​，从而去选择收益最大的特征。

# 优缺点

Pros

  1. 因为采用了二阶导，所以精度更高
  2. 只要支持一阶导数和二阶导数的CART和线性分类器都可以作为基学习器，比较灵活

  6. 支持缺失值的处理
  7. 可以并行化计算

Cons

  1. 计算量依然很大，需要遍历数据集
  2. 预先排序，不仅要存储特征，还得存储特征对应样本的梯度统计索引，空间开销大。
