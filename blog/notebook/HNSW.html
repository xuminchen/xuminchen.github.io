# HNSW

HNSW算法是为了能在庞大的数据集中，做高效率的相似度查找。

HNSW是由朴素查找法一步步演变过来的。

# 朴素查找法

方法就是：将点连在一起构成一个图，从任意一点出发，查找友点中哪个与目标点最近，则选择哪个点作为下一跳的点。

但是有一些缺点：

  1. 无友点的点无法被查询到
  2. 最近的两个点之间无连线的话会影响查询效率
  3. 若有一个点有多个友点，重复计算

因此，我们要求构图时每个点都要有友点，点与点之间都有连线，且要保证图中的线最少。因此，在NSW引入了德劳内法则。

# NSW

德劳内法则要求：

  1. 每个点都要有友点
  2. 相邻的点互为友点
  3. 图中连接数最少

但是德劳内法则的构建时间复杂度很高，查找效率并不能保证最好（因为和初始点相关，若初始点选的很差，那查找会比较慢），因此做了一些改进，引入了高速通道的思想。

方法如下：

向图中逐个随机插入点，插入后，通过朴素查找法的搜索思路，找到最近的m个点，并将插入点与这m个点相连作为友点。

# HNSW

HNSW是在NSW的基础上作了一些优化改进，提高查找效率。

首先是引入了跳表skip list的结构，表层作高速通道，底层作精细查找。

查找时，先计算当前点可以深入到第几层，计算公式：floor(−ln(uniform(0,1))∗ml)floor(-\ln(uniform(0,1))*ml)floor(−ln(uniform(0,1))∗ml)，ml是标准化因子。自顶层向目标点的层数L进行逼近搜索，直到L+1层，每层寻找离当前点最近的一个点。自L层向底层逼近，每层寻找离目标点的层最近的K个点赋值到集合W。在W中选择与当前点最近的M个点作为友点相连。每个友点的连接数大于MmaxM_{max}M​max​​的话，则缩减友点（把最远的友点去掉）。
