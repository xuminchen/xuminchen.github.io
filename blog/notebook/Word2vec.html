# Word2vec

# 传统的文本表示方法

在传统的机器学习过程中，诞生出一些古老的文本表示，以将文本数据表示成机器可读可理解的方式。在之前提到过，有像one-hot representation、tf-idf这样的形式。但是这样的表示方法有很大的局限性，比如像空间所能表达词汇的大小、语义相关性等，而且效果与样本数有关，如果数量越多，那么局部泛化能力越强，反之越弱。因此引入了分布式假设的概念。而Word2vec则正是分布式假设的代表。

# Introduction

word2vec的核心思想是在同一文本中，离得越近的两个单词相似度越高。（words close to each other tend to have similar sematic meaning）。

word2vec实际上并不是一个模型，而是一个工具。包括了两个真正的词向量模型：skip-gram和CBOW。另外真正意义上并不是一个语言模型。

如果一个词向量模型训练的越好，那么预测的准确率就越高。

# Structure

模型结构如上图所示

CBOW是一个通过上下文单词去预测中心词的模型，而skip-gram正好相反，是一个通过中心词去预测上下文单词。正如上图所示，在Projection层和output/input层是分别含有两个不同的矩阵的，分别用U和V来表示。

为什么要用两个矩阵呢？以Skip-gram举例，目的是为了在计算的时候，区分不同的向量是表示中心词的向量还是表示的是上下文的向量。输入的单词是以one-hot的形式输入，然后通过input处的U矩阵，获取到表示该单词的分布式向量表示，之后再通过Projection层的V矩阵，预测出其他词的概率分布。CBOW也同理。

与之前的模型相比，word2vec的训练所需的数据比以前更大，而且具有合适的维度。相似的词语表达有相似的语义规律性。

word2vec的目标就是增大准确率的同事，最小化计算带来的消耗。它的目标函数就是给出中心词后，出现上下文单词的最大概率，结合所有单词和所有上下文单词，有如下的数学表达:

# 损失函数

一般采用cross-entropy作为损失函数， 因为我们预测的是每个单词出现的概率分布，因此与目标的概率分布（0-1）比较，希望差距越小越好。

# 优化

如果用上述的方式，当词表规模一旦超过一定量级，那么时间复杂度和计算开销将会指数级增大。因此作者在一篇新的论文中，讲述了一些训练的trick，去提高训练的效率。

# Negative Sampling 负采样

为了优化训练，我们可以将问题用另一个问题来表示一下：P(1∣w,c;θ)P(1|w,c;\theta)P(1∣w,c;θ)，即判断该上下文单词是否与中心词同时出现（挨在一起），1代表同时出现，0代表没有同时出现。这样就把原本的问题转换成一个二分类的问题，P(1∣w,c;θ)=11+exp(−Uw∗Vc)P(1|w,c;\theta) = \frac{1}{1+exp(-U_w*V_c)}P(1∣w,c;θ)=​1+exp(−U​w​​∗V​c​​)​​1​​和P(0∣w,c;θ)=−exp(−Uw∗Vc)1+exp(−Uw∗Vc)P(0|w,c;\theta) = \frac{-exp(-U_w*V_c)}{1+exp(-U_w*V_c)}P(0∣w,c;θ)=​1+exp(−U​w​​∗V​c​​)​​−exp(−U​w​​∗V​c​​)​​。

此时目标函数可以更新为

这表示的是，我们希望若两个词挨在一起的时候，增大前一个部分（正样本）的概率，同时，如果两个单词不是在一起的，减小后面一部分（负样本）的概率。

但是，实际上真正挨在一起的单词（正样本）是很少的，而不挨在一起的单词也就是负样本却又很多组，∑(w,c)∈D^logσ(−uw⋅vc)\sum_{(w,c) \in \hat{D}} \log \sigma(-u_w \cdot v_c)∑​(w,c)∈​D​^​​​​logσ(−u​w​​⋅v​c​​)这一部分的计算开销太大了，时间复杂度为 O(∣V∣2)O(|V|^2)O(∣V∣​2​​)。在样本数很大的时候，这就导致了问题。因此提出了负采样的概念。

即，只考虑部分的负样本，并将其余的丢弃。

所以，我们进一步优化，仅仅考虑与当前中心词w有关的负样本：

那么，如何选取负样本呢？

  1. 根据文本中单词的词频采样：常出现的词在负采样中出现的次数越多，模型会经常做无谓训练

  2. 均匀分布采样：但是这样采样的词语并没有代表性

# Hierachical Softmax

在最后一层采用了哈夫曼树的结构，把一句话里面单词出现的次数进行记录并进行编码。

构建规则：每一次都从当前记录的值里面选取最小的两个权值去构建一棵字数。在构建完成之后，每个词出现次数都作为叶子结点。这些叶子结点的性质是度数（degree）都是1，即与当前节点相关联的点的个数是1.

那么有一个问题，为什么要选取最小的两个权值呢？因为要把所有词频进行编码，而我们希望高频词的编码个数少，低频词编码个数多，这也是构建哈夫曼树的思路，这样能够采用最少的长度把所有长度的单词进行编码。

有几点要注意：

  1. 哈夫曼树一定是二叉树
  2. 构建的规则固定，即一开始便要缺点左子树是1，右子树是0或左子树是0，右子树是1。
  3. 目的是找到最短的从根到叶子结点的路径。
  4. 代码中是没处理一个词，就更新v矩阵。

这里的判断左子树还是右子树的规则是用逻辑回归判断，里面参数是当前节点的词向量v和参数θ\thetaθ。

# Evaluation

如何评估训练好的模型的优劣呢？

一般从以下三个角度去评估：

  1. 在二维空间上可视化，来判断两个点的相近程度
  2. 求两个向量的相识度
  3. 通过类比关系来测量距离。（像论文中表示的，两个单词之间的图片）

# Drawback

尽管word2vec的普及和适用程度非常高，但是依然存在一些缺点。

Drawback Solution
严格意义上没有考虑上下文（一定程度上考虑了，预测周边词和预测中间词） 结合上下文考虑的模型，如ELMo和Bert
共现窗口的长度有限 用时序相关的语言模型，如RNN、LSTM
缺乏全局性，仅仅考虑到窗口内的词 如FM这样的模型，提取特征
oov和低频词的处理 像fasttext这样采用subword的结构
可解释性不够 找到一个更低维的空间。

# Question

word2vec对oov的处理？

将oov单词用UNK表示，但是每个UNK表示的是同样的向量。

最终采用哪个矩阵?

skip-gram中，一般采用的是前面的那个矩阵。但是在Richard Socher的实验结果中表名，U+V作为词向量表达的结果更好。
