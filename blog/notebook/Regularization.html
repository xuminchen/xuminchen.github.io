# Regularization

# Introduction

复杂度高的模型会导致过拟合的现象，所谓过拟合就是模型对训练集的拟合程度高，但是对测试集的拟合程度差。与之相反的是欠拟合的现象，就是对训练集的样本拟合程度差。

以下是导致过拟合的现象以及解决的方法

Reason Solution
模型的复杂度高 使用更简单的模型
参数个数多 减少参数
模型参数空间的选择 通过正则化，filter掉针对复杂空间的参数
样本数量少 增加样本数据

另外，还有欠拟合现象的原因及解决方法

Reason Solution
模型复杂度低 使用更复杂的模型
参数个数少 增加模型参数
正则化超参数过大 减少正则化参数

# 正则的形式

L=objective+λ∗penalityL = objective +\lambda * penalityL=objective+λ∗penality

penality就是正则项

L1正则：

L=objective+λ∗∑∣w∣L = objective + \lambda*\sum |w|L=objective+λ∗∑∣w∣

L2正则：

L=objective+λ∗∑w2L=objective + \lambda*\sum w^2L=objective+λ∗∑w​2​​

以上两种都是我们常用正则，但是具体使用哪一种，就根据我们的情况决定，λ\lambdaλ是一个超参数，相当于告诉模型应该更关注objective的部分还是penality的部分。

  • 当λ=0\lambda = 0λ=0，没有任何限制
  • 当λ\lambdaλ比较大，即是让w变得更小
  • 当λ\lambdaλ比较小，即是让w变得更大

所以λ\lambdaλ气到了一个平衡的作用。

# 常规正则

  1. L0 norm，相当于计算向量中有多少个非零参数的个数，目标函数相当于最小化非零数的个数。
  2. L1 norm，取绝对值
  3. L2 norm，第二范式，取平方
  4. Nuclear Norm， 相当于去限制矩阵的秩rank

# L1 & L2

L1和L2的共同点：都是使www变小，从而达到控制模型复杂度的目的。

但是不同的是，L1的特点是sparse，稀疏性。L1正则化会使很多较小的参数设置成0（起到一个特征选择的作用），相当于把不好的参数过滤掉了，当一般遇上稀疏相关的问题时，我们会使用L1去找到一个比较稀疏的解。

而L2会保留这些较小的参数。另外L2会更稳定。
