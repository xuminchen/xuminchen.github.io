# Hidden Markov Model

# 时序模型&非时序模型

什么是时序模型呢？时序模型就是可以处理带有时间信息、具有不限长的特征的序列数据的模型，常见的时序模型包括了RNN、HMM、CRF等。

而非时序模型则是像图模型和CNN这样的，维度固定的模型。

# 什么是HMM？

首先要说的是HMM是一个隐变量模型，隐变量模型的X，也就是观测值的维度一般是要比Z（隐变量）的维度更高，所以隐变量模型也可以看做是一种降维的方法。且一定是Z→XZ\rightarrow XZ→X

HMM的三个问题:

  1. Inference，即是推导问题，从观测值X推导出隐变量Z。这个 问题我们采用Viterbi算法去求解。
  2. Parameters Estimation，参数估计问题，求出概率的转移矩阵，即状态转移矩阵Transition Matrix，发射矩阵 Emission Matrix，初始状态下每个隐变量出现的概率，我们用π\piπ表示。同时用AAA来表示状态转移矩阵，用BBB来表示发射矩阵。该问题我们用EMEMEM算法去求解。
  3. 求边缘概率，也就是求某个序列出现的概率。可以通过动态规划的算法去求解。

# 两个关键任务

所以求解以上三个问题，就是关于HMM的关键，包含在了HMM的两个关键任务中，分别是：

  • 给定θ\thetaθ，以及观测变量XXX，找出最合适的ZZZ
  • 估计θ\thetaθ的值。但是在这个任务里，包含了两种情况，分别是Complete case 和 Incomplete case。Complete case 就是已经知道了X、Z的情况下，去估计θ\thetaθ的值，这个情况下，我们可以用MLE的思想去求解。而Incomplete case则是只知道XXX而不知道ZZZ的值的情况下去估计，这时候我们就采用EM算法。

# F/B 算法

F/B算法是可以用来求解推到问题的算法，可以拆借为Forward Algorithm 和Backward Algorithm两部分。

用数学表示如下: F/B: P(Zk∣x)=P(Zk,x)P(x)∝P(Zk,x)=P(Zk,x1:k,xk+1:n)=P(Zk,x1:k)P(xk+1:n∣Zk,x1:k)P(Z_k|x) = \frac {P(Z_k,x)}{P(x)}\propto P(Z_k,x) = P(Z_k,x_{1:k},x_{k+1:n})=P(Z_k,x_{1:k})P(x_{k+1:n}|Z_k,x_{1:k})P(Z​k​​∣x)=​P(x)​​P(Z​k​​,x)​​∝P(Z​k​​,x)=P(Z​k​​,x​1:k​​,x​k+1:n​​)=P(Z​k​​,x​1:k​​)P(x​k+1:n​​∣Z​k​​,x​1:k​​)

因为根据马尔科夫链的性质，当前状态只与前一个状态有关，因此P(Zk∣x)=P(Zk,x1:k)P(xk+1:n∣Zk)P(Z_k|x) = P(Z_k,x_{1:k})P(x_{k+1:n}|Z_k)P(Z​k​​∣x)=P(Z​k​​,x​1:k​​)P(x​k+1:n​​∣Z​k​​)

Forward: ak(x)=P(Zk,x1:k)a_k(x)=P(Z_k,x_{1:k})a​k​​(x)=P(Z​k​​,x​1:k​​)

Backward: bk(x)=P(xk+1:n∣Zk)b_k(x)=P(x_{k+1:n}|Z_k)b​k​​(x)=P(x​k+1:n​​∣Z​k​​)

参考网上一篇文章的句子：”前向-后向算法首先对于隐马尔科夫模型的参数进行一个初始的估计（这很可能是完全错误的），然后通过对于给定的数据评估这些参数的的价值并减少它们所引起的错误来重新修订这些HMM参数。从这个意义上讲，它是以一种梯度下降的形式寻找一种错误测度的最小值。之所以称其为前向-后向算法，主要是因为对于网格中的每一个状态，它既计算到达此状态的“前向”概率（给定当前模型的近似估计），又计算生成此模型最终状态的“后向”概率（给定当前模型的近似估计）。 这些都可以通过利用递归进行有利地计算，就像我们已经看到的。可以通过利用近似的HMM模型参数来提高这些中间概率进行调整，而这些调整又形成了前向-后向算法迭代的基础。“ -- reference: https://blog.csdn.net/romance89/article/details/73089412

总而言之,前向-后向算法的关键都是将问题拆分成子问题求解。

前向算法：用动态规划

P(Zk,x1:k)=∑Zk−1P(Zk,Zk−1,x1:k−1,xk)=∑Zk−1P(Zk−1,x1:k−1)P(Zk∣Zk−1,x1:k−1)P(xk∣Zk,x1:k−1,Zk−1)=∑Zk−1P(Zk−1,x1:k−1)P(Zk∣Zk−1)P(xk∣Zk)\begin{aligned} P(Z_k,x_{1:k}) &= \sum_{Z_{k-1}}P(Z_k,Z_{k-1},x_{1:k-1},x_k) \\ &= \sum_{Z_{k-1}}P(Z_{k-1},x_{1:k-1})P(Z_k|Z_{k-1},x_{1:k-1})P(x_k|Z_k,x_{1:k-1},Z_{k-1}) \\ &= \sum_{Z_{k-1}}P(Z_{k-1},x_{1:k-1})P(Z_k|Z_{k-1})P(x_k|Z_k) \end{aligned} ​P(Z​k​​,x​1:k​​)​​​​​=​Z​k−1​​​∑​​P(Z​k​​,Z​k−1​​,x​1:k−1​​,x​k​​)​=​Z​k−1​​​∑​​P(Z​k−1​​,x​1:k−1​​)P(Z​k​​∣Z​k−1​​,x​1:k−1​​)P(x​k​​∣Z​k​​,x​1:k−1​​,Z​k−1​​)​=​Z​k−1​​​∑​​P(Z​k−1​​,x​1:k−1​​)P(Z​k​​∣Z​k−1​​)P(x​k​​∣Z​k​​)​​

以这种方式，将问题拆分成了前一个时刻的子问题。

同理，后向算法:

P(xk+1:n∣Zk)=∑Zk−1P(Xk+1:n,Zk+1∣Zk)=∑Zk−1P(xk+2:n∣xk+1,Zk+1,Zk)P(xk+1∣Zk,Zk+1)P(Zk+1∣Zk)=∑Zk−1P(xk+2:n∣Zk+1)P(xk+1∣Zk)\begin{aligned} P(x_{k+1:n}|Z_k) &=\sum_{Z_{k-1}}P(X_{k+1:n},Z_{k+1}|Z_k)\\ &= \sum_{Z_{k-1}}P(x_{k+2:n}|x_{k+1},Z_{k+1},Z_k)P(x_{k+1}|Z_k,Z_{k+1})P(Z_{k+1}|Z_k)\\ &=\sum_{Z_{k-1}}P(x_{k+2:n}|Z_{k+1})P(x_{k+1}|Z_k) \end{aligned} ​P(x​k+1:n​​∣Z​k​​)​​​​​=​Z​k−1​​​∑​​P(X​k+1:n​​,Z​k+1​​∣Z​k​​)​=​Z​k−1​​​∑​​P(x​k+2:n​​∣x​k+1​​,Z​k+1​​,Z​k​​)P(x​k+1​​∣Z​k​​,Z​k+1​​)P(Z​k+1​​∣Z​k​​)​=​Z​k−1​​​∑​​P(x​k+2:n​​∣Z​k+1​​)P(x​k+1​​∣Z​k​​)​​

这样，就可以拆分成后续时刻最大观测序列的子问题。

在Complete Case中，用F/B算法很容易就可以求出了我们需要的参数。那么在Incompletecase中呢？我们是不知道z是什么的，所以我们也无法直接通过F/B去求解，那么该怎么办呢？这时候就要靠EM算法了。

# EM算法

在不知道Z的情况下，通过EM算法，可以去近似参数。

再次用数学公式描述一下两种情况:

  1. Complete case

l(θ;D)=logP(x,z∣θ)=logP(x∣z;θ)+logP(z∣θ)\begin{aligned} l(\theta;D) &= \log P(x,z|\theta)\\ &=\log P(x|z;\theta)+\log P(z|\theta) \end{aligned} ​l(θ;D)​​​​=logP(x,z∣θ)​=logP(x∣z;θ)+logP(z∣θ)​​

  1. Incomplete case

l(θ;D)=log∑ZP(x,z∣θ)=log∑ZP(x∣z;θZ)P(z∣θZ)\begin{aligned} l(\theta;D) &= \log \sum_Z P(x,z|\theta)\\ &=\log \sum_Z P(x|z;\theta_Z)P(z|\theta_Z) \end{aligned} ​l(θ;D)​​​​=log​Z​∑​​P(x,z∣θ)​=log​Z​∑​​P(x∣z;θ​Z​​)P(z∣θ​Z​​)​​

EM算法是严格增大的，结果是一定会收敛，另外EM不是找到global optimization的，而是找到一个local optimization，每次总会找到一个比当前更好的参数值。

EM算法包括了两个step: E-step & M-step，E代表的是Expectation，M代表Maximization，也就是我们要先找到一个我们想要得到的期望之后，再去最大化我们的参数。

而我们怎么用EM算法去求解我们想得到的参数呢？

很简单，就是先用E-step去求Z，通过联合概率分布，去逼近Z的值后，再通过M-step去求θ\thetaθ。

# EM for parameter estimation
while not coverage:
  compute_z_expectation()
  update_pi()

总的来说，就是先求 π\piπ，然后再求B， 最后再求A.
