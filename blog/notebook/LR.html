# Logistic Regression

# Introduction

逻辑回归是可以用来作二分类和多分类任务的机器学习模型。不仅仅是对NLP，对其他AI的研究方向来说，也是一个很好的baseline。

机器学习可以这样表达：学习一个f:x→yf:x \rightarrow yf:x→y的映射，即输入x到输出y的映射。

逻辑回归是一个判别模型，求的是一个条件概率P(Y∣X)P(Y|X)P(Y∣X)

# 如何推导

# 为什么不可以用线性回归来表示？

用线性回归的方式表示，即是P(Y∣X)=WTx+bP(Y|X)=W^Tx+bP(Y∣X)=W​T​​x+b，从概率的角度去思考，P(Y∣X)P(Y|X)P(Y∣X)是一个条件概率，需要满足概率的性质，即取值在[0,1][0,1][0,1]区间内，另外还有一点就是要求∑P(Y∣X)=1\sum P(Y|X)=1∑P(Y∣X)=1，如果第一点不满足，就更不用说这一点了。

既然上述理由不满足，那么换个思路，如果可以吧WTx+bW^Tx+bW​T​​x+b这个属于[−∞,+∞][-\infty,+\infty][−∞,+∞]区间的值映射到[0,1][0,1][0,1]中，是否就可以当做条件概率去看待呢？

这里引入逻辑函数Logistic Function，也就是我们常用的Sigmoid Function，输入x的范围是[−∞,+∞][-\infty,+\infty][−∞,+∞]，输出y的值域是[0,1][0,1][0,1]，数学表达是y=11+e−xy = \frac {1}{1+e^{-x}}y=​1+e​−x​​​​1​​，图像如下：

# 参数更新

用梯度下降法实现逻辑回归的梯度更新：

分别对w,bw,bw,b求导，然后不断迭代，直至收敛。

但是在实际的场景中，数据量是巨大的，这个时间复杂度是随着样本的个数线性增长的，这会导致效率非常低。

因此提出了SGD随机梯度下降法。

随机梯度下降法是每次更新计算一个样本的梯度，然后再通过这个样本的参数去更新模型的参数。

for iter = 1,2 ,...,T
	Shuffle({x,y})
	for i = 1,2,...,n
		new_w = w - a * delta_w
		new_b = b - a * delta_b

在随机梯度下降法中，步长，也就是learning rate会比在GD中的笑，因为采用SGD的时候，会引入一些噪声，减少补偿也是为了达到降噪的目的。

折中考虑，提出了mini-batch GD，一次性考虑多个样本，又不至于引入了过多的降噪。而且也比较稳定。

# 怎么判断逻辑回归是一个线性的分类器？

判断的标准是决策边界Decision Boundary，即落在决策边界两边的点的概率是等同的，各占1/2。

证明如下：

P(y=1∣x;w)P(y=0∣x;w)=1\frac {P(y=1|x;w)}{P(y=0|x;w)}=1​P(y=0∣x;w)​​P(y=1∣x;w)​​=1

对上式展开，即1e−(WTx+b)=1\frac{1}{e^{-(W^Tx+b)}}=1​e​−(W​T​​x+b)​​​​1​​=1

对等式的两边加对数，log1e−(WTx+b)=−loge−(WTx+b)=log1=0\log \frac{1}{e^{-(W^Tx+b)}}=- \log e^{-(W^Tx+b)}=\log 1=0log​e​−(W​T​​x+b)​​​​1​​=−loge​−(W​T​​x+b)​​=log1=0，所以WTx+b=0W^Tx+b=0W​T​​x+b=0，这就是LR的Decision Boundary。

# 为什么当给定的数据线性可分的时候，逻辑回归的参数会趋于无穷大？

因为当数据线性可分的时候，空间内存在无数个将数据切分开来的超平面，而这些超平面由不同的参数组合而成，因此会趋于无穷大。所以实际使用的过程中，LR会结合正则化方法使用。
